# FFI Design: C ABI for the Dart Binding

Status: **decided, not yet implemented**. This document records the FFI-layer decisions for
the future Dart binding (the successor to swisseph.dart) so they survive until that work
starts. The decisions below are made; the FFI crate itself is created when the Dart work
begins.

The premise the whole design rests on — `Ephemeris: Send + Sync`, no interior mutability —
is enforced today by `tests/send_sync.rs` (compile-time assertion + 8-thread bitwise
determinism test).

## Decision: hand-rolled C ABI + cbindgen, in a separate `swisseph-ffi` crate

A dedicated `swisseph-ffi` workspace member (`crate-type = ["cdylib"]`), added when the
Dart work starts. It hand-defines `extern "C"` functions and `#[repr(C)]` mirror types,
with the C header generated by **cbindgen**. **NOT flutter_rust_bridge.**

Rationale:

- The API surface is flat numeric data — `f64` arrays, `i32` codes, UTF-8 names. There
  are no async streams, callbacks, or object graphs, so frb's codegen and runtime buy
  nothing here.
- The consumer (the swisseph.dart successor) already speaks `dart:ffi` against the C
  Swiss Ephemeris ABI. A hand-controlled ABI can mirror the shapes that binding already
  handles, making the migration a re-point rather than a rewrite.
- Sharing one `Ephemeris` across Dart isolates works by passing the raw pointer as an
  int (see handle model below). That requires a stable, auditable ABI surface where we
  control the handle's ownership semantics — not generated glue that owns the handle
  model itself.

Rejected alternative: **flutter_rust_bridge**. Revisit only if the Dart side ever needs
streams or async event search; for a pure request/response numeric API it is overhead
without payoff.

## Handle model

- `swisseph_new(config…) -> *mut Ephemeris` via `Box::into_raw`. Construction takes the
  flattened config parameters; the returned pointer is the opaque handle.
- Dart isolates share the instance by passing the pointer **as an int** — Dart cannot
  send objects across isolates, but ints travel freely. This is sound precisely because
  `Ephemeris` is `Send + Sync` with no interior mutability: any number of isolates may
  call into the same handle concurrently with zero synchronization and get bitwise-
  identical answers (the guarantee `tests/send_sync.rs` pins).
- `swisseph_free(*mut Ephemeris)` via `Box::from_raw`, registered with a Dart
  `NativeFinalizer` so the handle is reclaimed when the owning Dart object is collected.
- Config is **immutable after construction** — there are NO `set_*` mutators in the ABI.
  Per-call variance (topographic position, sidereal mode) rides as explicit call
  parameters mapping to the existing `*_with_config` pattern in the lib crate. This is
  what keeps concurrent isolate use race-free by construction rather than by locking.

## Error convention

- `i32` return codes: `0` = OK, negative = error — mirroring C Swiss Ephemeris'
  `retflag < 0` convention the Dart side already handles.
- An error-code enum mapped 1:1 from `swisseph::Error` variants.
- Results come back through out-params (see flattening table).
- Error messages: each fallible call takes a caller-allocated
  `char* err_buf, usize err_cap` out-param pair that receives the UTF-8 message on
  failure. **No thread-local or handle-global "last error"** — a stateless per-call
  buffer is isolate-safe by construction; a `last_error()` accessor would reintroduce
  exactly the shared mutable state the crate exists to avoid.

## Type flattening

| Rust type | C ABI shape |
|---|---|
| `CalcResult` | `double xx[6]` out-param + `int32_t* flags_used` out-param |
| enums (`Body`, flags, house systems…) | `int32_t` using the existing C constant values (`Body` already mirrors `ipl` numbering; flags are already C bit values) |
| `&str` star/planet names (in) | `const char*`, UTF-8, NUL-terminated |
| `String` names (out) | caller-allocated `char* buf, usize cap` |
| `Result<T, Error>` | `int32_t` return code + out-params; message via `err_buf` |
| `Ephemeris` | opaque `*mut` handle (never `#[repr(C)]`) |

## What the lib crate must uphold

This is the contract the FFI layer depends on, enforced by `tests/send_sync.rs`:

- `Ephemeris` and `EphemerisConfig` are `Send + Sync`.
- **No interior mutability is ever added to `Ephemeris`** — no `Cell`, `RefCell`,
  `Mutex`, `OnceCell`, lazy caches, nothing. If a cache is ever wanted, it must be a
  separate opt-in wrapper type, not a field on `Ephemeris`. (And never
  `unsafe impl Send/Sync` to paper over a violation — the assertion failing means the
  architecture broke, not that the test needs fixing.)
- `#[repr]` stability is NOT required on lib-crate types: `swisseph-ffi` defines its own
  `#[repr(C)]` mirrors and converts at the boundary. The lib crate stays idiomatic Rust.
